## 1. 项目概述

本项目参考 PetShop DApp 实验流程，实现运行在以太坊兼容本地链上的众筹系统。系统使用 Solidity 编写智能合约，使用 Hardhat 完成编译、测试和本地节点管理，使用 Hardhat Ignition 部署合约，使用 Ethers.js 与 MetaMask 完成前端交互。

系统形成“项目创建—用户捐赠—到期结项—成功提现或失败退款”的完整资金闭环，并增加里程碑放款、捐赠者投票、早期支持积分、纪念凭证、事件追踪和多页面前端等扩展功能。

## 2. 系统架构

系统分为四层：

1. 前端展示层：原生 HTML、CSS 和 JavaScript。
2. 交互层：Ethers.js 创建合约对象，MetaMask 管理账户和签名。
3. 合约层：`Crowdfunding.sol` 保存业务状态并执行资金规则。
4. 区块链层：Hardhat Local 提供 Chain ID 31337 的本地网络。

写操作的执行链为：网页参数校验、Ethers.js 构造交易、MetaMask 签名、交易广播、Hardhat 节点执行、合约修改状态、交易进入区块、前端等待回执并重新读取状态。只读查询不修改状态，不需要签名，也不消耗 Gas。

## 3. 核心数据设计

`Project` 结构保存项目 ID、发起人、名称、描述、分类、封面 URL、目标金额、截止时间、已筹金额、状态、里程碑进度、已释放金额和退款池等数据。

捐赠金额使用二维映射：

```solidity
mapping(uint256 => mapping(address => uint256)) public donations;
```

由于 Solidity 映射不能遍历，系统额外使用 `projectDonors` 数组保存每个项目的不同捐赠者地址，并使用 `hasDonated` 防止同一地址重复加入数组。

## 4. 基本功能实现

### 4.1 项目创建

`createProject()` 检查名称、分类、目标金额和截止时间，分配唯一项目 ID，将 `msg.sender` 保存为发起人，并把项目初始化为 `Active`。创建结果通过 `ProjectCreated` 事件记录。

### 4.2 资金捐赠

`donate()` 要求项目进行中、尚未截止且金额大于零。ETH 进入合约托管，函数累计项目已筹金额和个人捐赠金额，并维护捐赠者列表。

### 4.3 到期结项

项目不会自动执行。截止时间到达后，任意账户可调用 `endProject()`：

- `totalRaised >= goal`：状态变为 `Successful`；
- `totalRaised < goal`：状态变为 `Failed`。

### 4.4 成功提现

未启动里程碑模式的成功项目由发起人调用 `withdrawFunds()` 一次性提现。函数检查项目状态、发起人身份、重复提现和里程碑互斥规则，并按“检查—状态更新—外部转账”的顺序执行。

### 4.5 失败退款

失败项目由每名捐赠者自行调用 `refund()`。未发生阶段放款时可以退回全部捐赠；若已经释放里程碑资金，只按原始捐赠比例退还合约中的未释放资金。状态在转账前清零，并使用重入锁保护资金函数。

## 5. 扩展功能

### 5.1 里程碑放款

里程碑模式必须在项目成功结项前启动：

- 第一阶段要求达到目标的 25%；
- 第二阶段要求达到目标的 50%；
- 第三阶段要求达到目标的 75%；
- 第四阶段要求项目成功结项。

前三阶段每次固定释放 `goal / 4`，第四阶段释放 `totalRaised - releasedAmount`。每阶段必须获得捐赠者严格多数票，即 `floor(donorCount / 2) + 1`。同一地址每阶段只能投一票，但曾经捐赠过的地址可以参与后续阶段投票。

### 5.2 早期支持奖励

项目创建后最长 24 小时内，前 10 位不同的首次捐赠者获得 100 点积分。积分进入账户级余额，可在独立奖励中心消耗 100 点兑换纪念凭证并加入奖励白名单。

### 5.3 多页面与事件追踪

前端拆分为项目操作、系统总览、我的项目、项目详情、链上事件和奖励中心。事件页面展示事件类型、区块时间、区块高度和交易 Hash，支持按项目 ID 过滤并导出 JSON。

## 6. 安全设计

- 使用 `require` 检查权限、状态、截止时间和金额。
- 提现、退款和阶段释放使用重入锁。
- 外部转账前先更新内部状态，遵循 Checks-Effects-Interactions。
- 成功项目不能退款，失败项目不能提现。
- 一次性提现与里程碑模式互斥。
- 退款后清零个人记录，防止重复退款。
- 前端按钮禁用只改善体验，最终权限仍由合约强制保证。

## 7. 工程实现与测试

项目提供节点启动、Ignition 部署、网页启动、演示数据、链上时间推进、环境检查和 Gas 报告脚本。前端自动读取 Ignition 部署地址和 Hardhat ABI，并检查目标地址是否存在合约字节码。

自动化测试共 14 项，覆盖项目创建、捐赠、早期奖励、结项、一次性提现、失败退款、多数投票、四阶段释放、成功后禁止临时启动里程碑、按目标金额释放、未释放资金退款、积分兑换和项目取消。当前测试全部通过。

## 8. 总结

本项目完成了课程要求的全部基本功能，并通过明确的状态机和资金托管规则实现成功提现与失败退款。扩展的阶段放款和早期奖励机制增强了众筹系统的资金约束与用户激励。后续可继续接入 IPFS、公开测试网、标准 ERC-721、事件索引服务和更完整的安全审计。
